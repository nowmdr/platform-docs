# Промпт для агента репозитория сайта CozyCorner: SEO-посты (`posts.post_type`)

Скопируй текст ниже в сессию агента, работающего с репозиторием сайта cozycorner.

---

Задача: поддержать SEO-посты — посты, которые НЕ выводятся в ленте блога, но
доступны по прямому URL и попадают в sitemap (programmatic SEO под
long-tail-запросы; создаются админкой/агентом).

Контекст:
- В `cozycorner.posts` добавлена колонка `post_type` (`'blog'` | `'seo'`,
  default `'blog'`). Миграция УЖЕ применена к живой БД из админки (через
  Supabase MCP, имя `add_seo_post_type`) — в репозиторий сайта нужно добавить
  файл миграции с тем же содержимым, чтобы история миграций осталась полной:
  ```sql
  alter table cozycorner.posts
    add column if not exists post_type text not null default 'blog'
    check (post_type in ('blog', 'seo'));
  create index if not exists posts_post_type_idx on cozycorner.posts (post_type);
  ```
  `if not exists` делает реплей на живой БД безопасным.
- Структура SEO-поста идентична блог-посту: те же секции
  `post_text_sections`/`post_product_sections`, тот же slug, `is_published`,
  `seo_title`/`seo_description`. Рендер страницы поста менять не нужно.

Что сделать:
1. **Лента блога** (список постов на `/blog` и любые другие места, где
   перечисляются посты — виджеты «последние посты», RSS, если есть): добавить
   фильтр `post_type = 'blog'` (в дополнение к существующему
   `is_published = true`).
2. **Страница поста** `/blog/[slug]`: фильтр по `post_type` НЕ добавлять —
   пост любого типа должен открываться и рендериться одинаково по прямому URL.
3. **Sitemap**: включать ВСЕ published-посты обоих типов (`blog` и `seo`) —
   sitemap — основной способ, которым поисковики находят SEO-посты.
4. **ВАЖНО — чего НЕ делать** (иначе фича теряет смысл или превращается в
   cloaking с риском санкций Google на весь домен):
   - НИКАКИХ редиректов для SEO-постов: человек, открывший прямой URL, видит
     обычную статью;
   - НИКАКОЙ логики по User-Agent (различать ботов и людей нельзя);
   - НЕ добавлять `noindex`/`nofollow` на страницы SEO-постов;
   - canonical у SEO-поста указывает на самого себя (как у обычного поста).

Критерии приёмки:
- Пост с `post_type='seo'` (published) отсутствует в ленте `/blog`, но
  открывается по `/blog/<его-slug>` и есть в sitemap.
- Существующие блог-посты в ленте — без изменений.
- Мета-теги SEO-поста (`seo_title`/`seo_description` с фолбэком на
  title/excerpt) работают так же, как у блог-постов.
