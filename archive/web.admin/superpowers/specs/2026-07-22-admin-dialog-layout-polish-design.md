# Спека: полиш layout + унификация модалок (web.admin)

> Date: 2026-07-22 | Project: web.admin | Scope: только фронт, без БД
> Related: [components.md](../../../../admin-panel/components.md), [products.md](../../../../admin-panel/products.md)

Чисто фронтовая работа в `web.admin`: расширить основной контейнер, починить два
визуальных бага в пикерах и унифицировать диалоги вокруг одного примитива с
предсказуемой высотой и скроллом. Часть про Category/Brand — отдельная спека
(`2026-07-22-category-brand-unification-design.md`).

## Цель и критерии успеха

- Контент админки использует больше ширины экрана (+~25%).
- В пикере товаров скроллбар не перекрывает чек-индикаторы строк.
- В пикере картинок выделение текущей картинки заметное, мягкое и не обрезается.
- Все модалки строятся из одного примитива: фиксированные header/footer,
  скроллится только середина, высота ограничена сверху (`max-h`), снизу — по
  контенту; ручные `max-h-[NNsvh]` на внутренних div убраны.

Вне scope: любые изменения БД, раздел Category/Brand, редизайн содержимого модалок.

## 1. Ширина основного контейнера

`src/components/AppShell.tsx`: заменить `max-w-5xl` → `max-w-7xl` (64rem → 80rem,
+25%) в двух местах синхронно:
- шапка — строка 19 (`div.mx-auto … max-w-5xl`);
- `<main>` — строка 33 (`main.mx-auto max-w-5xl p-4`).

Оба меняем вместе, чтобы шапка и контент оставались выровнены по одной сетке.
Обновить ориентир «Контент страницы — `max-w-5xl`» в
[components.md §5](../../../../admin-panel/components.md) и §3.3 (шапка `max-w-5xl`).

Проверка: страницы (Media/Products/Blog/Pages) шире; двухколоночный SiteLayout
(меню `w-44` + контент `flex-1`) не ломается — меню фикс, растёт правая колонка.

## 2. Скроллбар в пикере товаров (ProductPickerDialog)

`src/features/posts/ProductPickerDialog.tsx`, скролл-контейнер (сейчас
`div.max-h-[50svh] overflow-y-auto`, строка 88): скроллбар перекрывает круглый
чек-индикатор строки (`span.size-5 … rounded-full`, справа в кнопке).

Фикс: правый внутренний отступ на скролл-области, чтобы контент строк отступал от
жёлоба скроллбара (`pr-1`, при необходимости `pr-2`). После перевода на `DialogBody`
(см. §4) паддинг задаём на нём или на `<ul>`.

Проверка: при 10+ товарах и активном скролле кружок-галочка полностью виден, между
ним и скроллбаром есть зазор.

## 3. Обводка выбранной картинки (ImagePickerDialog)

`src/features/products/ImagePickerDialog.tsx`:
- Кнопка плитки (строка ~209): текущая картинка помечена `ring-2 ring-ring` — серое
  малозаметное кольцо. Заменить на заметное, но не резкое:
  `ring-2 ring-primary ring-offset-2 ring-offset-background`.
- Скролл-контейнер грида (сейчас `div.max-h-[55svh] overflow-y-auto`, строка 180)
  обрезает кольцо у крайних плиток (кольцо рисуется за границами кнопки, паддинга
  нет). Дать внутренний паддинг `p-1` (после перевода на `DialogBody` — на самом
  `DialogBody` или гриде), чтобы кольцо не срезалось краем скролл-области.

Проверка: выбранная картинка в любом углу грида имеет полное неусечённое кольцо
цвета primary; при скролле кольцо не липнет к краю.

## 4. Унификация диалогов (ui/dialog.tsx + DialogBody)

Сейчас `DialogContent` (`src/components/ui/dialog.tsx`) — `grid gap-4` без
ограничения высоты; каждая модалка сама навешивает `max-h-[50svh]`/`max-h-[55svh]`
на внутренний скролл-div. Унифицируем в самом примитиве.

Решение (выбрано пользователем: расширяем `ui/dialog.tsx`, файл уже кастомизирован —
bespoke footer, `showCloseButton`):

1. **`DialogContent`**: раскладку `grid` → `flex flex-col`, добавить `max-h-[85svh]`.
   Паддинг `p-4` и негативные отступы футера (`-mx-4 -mb-4`) в flex-колонке
   работают как прежде. `min-h` не задаём — минимальная высота по контенту.
2. **Новый экспорт `DialogBody`** — скроллящаяся середина:
   `min-h-0 flex-1 overflow-y-auto -mx-4 px-4` (плюс место под §2/§3 паддинги:
   право/`p-1`). `min-h-0` обязателен — иначе flex-ребёнок не сожмётся и скролл не
   появится. Негативный `-mx-4 px-4` тянет фон/скролл на всю ширину контента, но
   сохраняет внутренний отступ (кольца и чек-индикаторы не режутся).
3. Экспортировать `DialogBody` рядом с остальными в `dialog.tsx`.

Раскладка любой модалки после унификации: `DialogHeader` (фикс) → `DialogBody`
(скролл, `flex-1`) → `DialogFooter` (фикс, опционально). Высота: минимум по
контенту, максимум `85svh`, при переполнении скроллится только тело.

### Рефактор потребителей

- **ProductPickerDialog** (`features/posts/`): поиск + `DialogBody` со списком (убрать
  `div.max-h-[50svh] overflow-y-auto`), футер Cancel / Add N products. §2 паддинг — на
  теле/списке.
- **ImagePickerDialog** (`features/products/`): панель фильтр+поиск+Upload, `DialogBody`
  с гридом (убрать `div.max-h-[55svh] overflow-y-auto`), §3 паддинг+кольцо.
- **MediaDetailsDialog** (`features/media/`): проверить, что 3:2-грид с превью и
  метаданными помещается в новую схему; если контент длинный — завернуть в
  `DialogBody`. (Прочитать файл перед правкой — раскладка нетривиальная.)

Что НЕ трогаем: `AlertDialog` (`ui/alert-dialog.tsx`) — отдельный примитив, у него
свои короткие подтверждения; в scope не входит.

## Проверка (перед завершением)

- `npm run build` и `npm run lint` (oxlint; 2 известных shadcn-warning — ок) в `web.admin`.
- Ручной прогон: открыть пикер товаров (много строк) — скролл не режет галочки;
  пикер картинок — кольцо заметное и целое в углах; ресайз окна — модалки не
  выходят за `85svh`, скроллится только середина; ширина страниц выросла и layout
  не поехал.

## Риски / заметки

- `ui/dialog.tsx` формально «CLI-only» по AGENTS.md, но уже кастомизирован; правка

  проверить, что апдейт dialog не затрёт `DialogBody`/`max-h`.
- `MediaDetailsDialog` — единственная модалка со сложной внутренней сеткой; для неё
  оценить, нужен ли скролл всего тела или только правой колонки метаданных.
