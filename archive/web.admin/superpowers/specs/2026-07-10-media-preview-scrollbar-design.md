# Медиа-модалка: стабильное превью + глобальный тонкий скроллбар — дизайн

Дата: 2026-07-10. Статус: утверждён пользователем (чат).

## 1. Превью в `MediaDetailsDialog`

Проблема: врапер превью без собственных габаритов — растягивается по высоте
правой колонки, маленькие картинки выглядят коряво.

Решение: врапер — квадрат на всю ширину левой колонки
(`aspect-square w-full self-start`, `bg-muted`, `border`, `rounded-lg`,
`overflow-hidden`, flex-центрирование). Картинка `max-h-full max-w-full
object-contain`: большая ужимается, маленькая — натуральный размер по центру
(без апскейла). `max-h-[70svh]` с врапера и картинки убирается (квадрат по
ширине колонки и так меньше).

## 2. Глобальный скроллбар (`src/index.css`, в `@layer base`)

- `html { scrollbar-gutter: stable; }` — место под скроллбар зарезервировано,
  появление/исчезновение скролла не меняет ширину контента.
- Минималистичный вид: `* { scrollbar-width: thin; scrollbar-color:
  var(--border) transparent; }` (Chrome 121+, Firefox); для Safari внутри
  `@supports not (scrollbar-color: auto)` — `::-webkit-scrollbar` 8px,
  прозрачный трек, скруглённый бегунок `var(--border)`, hover —
  `var(--muted-foreground)`. Действует и на внутренние скроллы (пикер, диалоги).

## Проверка

`npm run build` + `npm run lint`; глазами: маленькая (679×470) и большая
картинка в модалке — врапер одинаковой формы, картинка по центру; страницы с
коротким и длинным контентом — ширина не прыгает; вид скроллбара тонкий.
