# SEO Posts Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Раздел «SEO Posts» в админке — посты, идентичные блогу, но с `post_type='seo'`: не попадают в ленту блога на сайте, доступны по прямому URL и через sitemap; создаются агентом claude.ai через Supabase-коннектор.

**Architecture:** Колонка `post_type` в `cozycorner.posts` (default `'blog'`); `PostsPage`/`PostEditPage` параметризуются конфиг-объектом раздела (тип, базовый путь, секция папок, лейблы); свой набор папок `seo_posts` в `admin_folders`. Спека — `docs/superpowers/specs/2026-07-16-seo-posts-design.md`.

**Tech Stack:** Vite + React + TS, TanStack Query, react-router data router, Supabase (MCP для миграции).

**Проверка:** в репо нет тестового фреймворка — проверка `npm run lint` + `npm run build` + ручной e2e-чек-лист (паттерн всех планов проекта). НЕ коммитить без явного разрешения пользователя (CLAUDE.md).

- [ ] **Step 2.2: Фильтр в `listPosts`**

```ts
export async function listPosts(
  site: SiteConfig,
  postType: PostType,
): Promise<PostListItem[]> {
  const { data, error } = await getDb(site)
    .from("posts")
    .select("id,title,published_at,is_published,folder_id")
    .eq("post_type", postType)
    // tiebreaker: даты импортированных постов могут совпадать (паттерн products)
    .order("published_at", { ascending: false })
    .order("title", { ascending: true });
  if (error) throw error;
  return (data ?? []) as PostListItem[];
}
```

`getPost`/`createPost`/`updatePost`/`deletePost` — без изменений (тип едет в `PostInput`).

export function PostsPage({ section }: { section: PostsSectionConfig }) {
